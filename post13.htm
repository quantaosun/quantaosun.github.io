Gromacs 快速编译指南

Gromacs 是完全免费的主流分子模软件，在蛋白质，核酸和小分子模拟方面应用广泛。目前不支持在Windows系统直接安装使用。

本博客将尽量简短的介绍Gromacs的编译过程，请注意，这种编译为快速编译，得到的软件不一定能发挥安装电脑的最大计算能力。如果要做一个完美的

编译，将显著提升安装的难度,在此不做讨论，且本过程仅适用于个人电脑，不适合远程超算安装。

编译安装要求。平台―Unix,比如Ubuntu.

安装包： Gromacs 官方安装包。

请自行前往Gromacs 官方网站下载安装包，安装过程中的界面图片可参考我的另一篇关于gromacs 使用的博客。

现在以最新版本为例，安装包为gromacs-2021-beta2.tar.gz

运行以下指令：

a.  tar xfz gromacs-2021-beta3.tar.gz

b.  cd gromacs-2021-beta3

c.  mkdir build

d.  cd build

注意到目前位置不会有错误出现。

预警

e 指令可能会有错误出现。如无报错就继续跳跃到指令f.

e. cmake .. -DGMX_BUILD_OWN_FFTW=ON -DREGRESSIONTEST_DOWNLOAD=ON

这一步的错误可能有 1. 我们定义的-dregressiontest_download为ON，也就是在编译过程中

从一个云端的gromacs 站点下载一个必须的压缩文件，经过检测，2019.3 版本的下载过程是有问题的，这是

开发者维护的问题， 如果报错云端连接超时或者下载失败，建议直接更换安装别的版本。你也可以从第三方来源

手动下载这个regressiontest.tar.gz, 然后将e指令的ON, 改为OFF，并提供手动下载包的路径，具体安装包的名称，下载来源

以及如何定义手动安装包的方法请自行搜索。

第二种错误就是，对于那些新安装的ubuntu等系统，容易报错“cmake”缺失或者报错C++ 编译环境路径没有定义，针对这种报错，请按照下面的

1-5 步骤排错。

1. sudo apt update

2.sudo apt upgrade

请注意，1，2 为命令行输入，下面的3，4 为图形界面的软件商店步骤。

3. Go to ubuntu Appstore (I don't recommend you use the command line which will be tedious)

步骤3，4 也可在命令行完成，但本着怎么简单怎么来的精神，最快的方法，最容易让其他人明白的方法是从ubuntu的软件商店安装cmake.

接下来的命令5，返回命令行执行。

4. In the search bar, search cmake, and install it

第五步不要忘记，这一步系统不会主动提示，但是少了会出现“C++ 编译路径没有定义“等报错。

5. sudo apt-get install build-essential

f. make

g. make check

h. sudo make install

i. source /usr/local/gromacs/bin/GMXRC

最后请打开 ~/.bashrc 文件，在最后一行写入gmx 的环境变量。否则下次，当你新打开一个

新的命令窗口输入gmx 时，系统将无法识别或者建议你重新安装gmx, 不要重新安装！

echo "source /usr/local/gromacs/bin/GMXRC" >> ~/.bashrc

安装完成后，输入gmx, 系统将会识别并返回帮助信息，则安装成功。

以上的g 步骤过程可能会有一两个失败的，比如FEPtest 失败等，但是只要绝大多数测试通过，比如70个测试中69个通过，你可以忽略

报错的情况，那可能是由于显卡加速没有设置到位，但因为个人电脑也不指望什么显卡加速了，如果真的有大的体系，和长时间的模拟要

跑，请使用超算版本。所以对于10 ns 一下的普通大小的蛋白质模拟，没有必要设置的所有东西都是完美的，因为你就如一开始所说，如果你

 

追求完美的安装，你所花的时间来学习安装这些IT知识的时间将显著升，毕竟使用gromacs 的大多是生物，

化学背景的人，关注的重点在于如何使用 Gromacs 解决本学科的难题，一味追求完美的工具，将迷失在无边的代码中！！！

This is a short tutorial of how to compile Gromacs, a-i, is the installation steps,

and 1-5 is the method to get rid of an error of missing cmake. If you don't get any error r

during the compile process, you can skip step1-5, but if you encounter a problem at step e, please

come back to run1-5 in order.

 

Also, it has been noticed that version Gromacs/2019.3 installation is problematic due to the

remote server of gromacs ftp service, so if you encounter some weird error like some necessary

files can't be accessed during the compiling process, maybe just consider to switch to another

version and try again. As you could tell, the compiling process needs an internet connection.

 

The whole process may take up to one hour even everything is correct, depending on your hardware and internet connection speed.

 

a.  tar xfz gromacs-2021-beta3.tar.gz

b.  cd gromacs-2021-beta3

c.  mkdir build

d.  cd build

e. cmake .. -DGMX_BUILD_OWN_FFTW=ON -DREGRESSIONTEST_DOWNLOAD=ON

f. make

g. make check

h. sudo make install

i. source /usr/local/gromacs/bin/GMXRC

At last, as always, remember to write the souce line to /.bashrc

echo "source /usr/local/gromacs/bin/GMXRC" >> ~/.bashrc

 

Now, a very nasty error, especially you just installed a ubuntu system, "cmake is missing", the system will suggest you "sudo apt install cmake"

The system suggestion is not wrong but just not enough to help you out of that problem.

If you do like the suggestion, you may solve it, but for others, there might be another

error remaining about C++ compiler path not defined, etc.

So, I suggest you do like 1-5 as per below, then you have a great chance to have no error.

1. sudo apt update

2.sudo apt upgrade

3. Go to ubuntu Appstore (I don't recommend you use the command line which will be tedious)

4. In the search bar, search cmake, and install it!

5. sudo apt-get install build-essential

e. cmake .. -DGMX_BUILD_OWN_FFTW=ON -DREGRESSIONTEST_DOWNLOAD=ON

f. make

g. make check

h. sudo make install

i. source /usr/local/gromacs/bin/GMXRC

At last, as always, remember to write the souce line to /.bashrc

echo "source /usr/local/gromacs/bin/GMXRC" >> ~/.bashrc

Good luck!
